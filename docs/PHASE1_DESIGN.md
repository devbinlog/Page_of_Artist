# Phase 1 — Design Interview Summary
# Page of Artist

> 설계 인터뷰 완료일: 2026-03-16

---

## 프로젝트 개요

- **프로젝트명**: Page of Artist
- **목표**: 아티스트 정보를 3D 인터랙티브 씬으로 표현하는 포트폴리오 프로젝트
- **타겟 플랫폼**: 모바일 WebView + 데스크탑 브라우저
- **기술 스택**: React, TypeScript, Three.js, React Three Fiber, @react-three/drei

---

## 페이지 구조

```
로딩 페이지 → 인트로 페이지 → 갤러리 페이지 → 아티스트 페이지
```

---

## 페이지별 상세 설계

### 1. 로딩 페이지

- Spotify API 데이터 fetch 동안 표시
- 3D 비닐 레코드 회전 애니메이션
- 레코드 외곽 테두리가 로딩 진행률에 따라 채워짐
- 사용자 인터랙션: 드래그로 레코드 회전 속도 조작 가능
- 로딩 완료 시 인트로 페이지로 자동 전환

---

### 2. 인트로 페이지

- 모든 UI 요소가 3D 오브젝트로 구성
- **구성 요소**
  - 프로젝트 소개 텍스트 (3D)
  - 검색창 (3D) — 아티스트 이름 또는 장르로 검색
  - 검색 결과 — 텍스트 목록 형태, 클릭 시 해당 아티스트 페이지로 이동
  - 갤러리 이동 버튼 (3D)
- **배경 요소**
  - 음표, 조표 등 3D 입체 오브젝트가 자유롭게 소수 부유
  - 오선지: 화면 하단 30px 위에 고정 배치

---

### 3. 갤러리 페이지

- **레이아웃**: 원형 캐러셀
- **카메라**: 고정, 카드들이 원형으로 회전
- **노출 카드 수**: 한 번에 8개
- **전체 아티스트 수**: 30명, 랜덤 순서 배치
- **카드 상태**: 약간 둥둥 떠있는 부유 애니메이션

#### 네비게이션
| 동작 | 결과 |
|------|------|
| 마우스 오른쪽으로 흔들기 | 다음 카드 |
| 마우스 왼쪽으로 흔들기 | 이전 카드 (초기 상태에서는 새 카드) |
| 자동 넘김 | 몇 초마다 자동 전환, 전환 애니메이션 0.3s |

#### 카드 선택 흐름
1. 카드 **클릭** → 카드가 화면 중앙으로 이동 + 확대
2. 중앙 확대 상태에서 마우스 좌우/상하 흔들기 → 카드 3D 틸트 확인
3. **더블클릭** → 카드가 점점 확대되는 줌 트랜지션 → 아티스트 페이지로 이동

---

### 4. 아티스트 페이지

- 카드 오브젝트 중앙 배치
- 링크 큐브 3개가 카드 주변에서 제자리 부유

---

## 핵심 오브젝트 설계

### 카드 오브젝트

#### 앞면 레이어 구조 (앞 → 뒤)

| 순서 | 요소 | 비고 |
|------|------|------|
| 1 | 아티스트 대표 이미지 | 최전면 |
| 2 | 아티스트 이름 텍스트 | |
| 3 | 장르 태그 | |
| 4 | 대표곡 이름 + YouTube 링크 | 수동 입력 데이터 (Spotify API 비연동) |
| 5 | 앨범 기반 그라디언트 배경 | 반투명, 최후면 |

#### 뒷면 구조

| 순서 | 요소 | 비고 |
|------|------|------|
| 1 | 앨범명 | 클릭 시 YouTube 이동 (수동 입력 데이터) |
| 2 | 트랙 리스트 (번호 순 나열) | Spotify API 데이터 |
| 배경 | 앨범 기반 그라디언트 | 앞면과 동일 스타일 |

#### 3D 효과

- **틸트 + 패럴랙스 혼합 (C타입)**
  - 카드 전체가 마우스 방향으로 기울어짐
  - 내부 레이어들이 각기 다른 깊이로 움직임
- **앞뒤 전환**
  - 클릭 시 Y축 기준 천천히 회전하여 앞/뒤 전환

---

### 링크 큐브 오브젝트

- **형태**: 3D 큐브에 로고 아이콘 부착
- **종류**: YouTube, Spotify, Instagram
- **데이터**: YouTube/Instagram 링크는 수동 입력, Spotify 링크는 API 제공 `external_urls.spotify` 사용

#### 인터랙션

| 동작 | 결과 |
|------|------|
| 기본 상태 | 제자리에서 위아래 둥둥 부유 |
| 호버 | 살짝 확대 + 테두리 빛 발광 효과 |
| 더블클릭 | 해당 링크로 이동 |

---

## 씬 분위기 및 공통 요소

### 배경
- 색상: 흰색 + 매우 약한 하늘색 tint
- 조명: 미니멀

### 배경 음악 요소
| 요소 | 형태 | 동작 |
|------|------|------|
| 오선지 | 2D 고정 | 화면 하단 30px 위 고정 |
| 음표, 조표 등 | 3D 입체 | 자유롭게 소수 둥둥 부유 |

### 카메라
- 마우스 방향으로 미세하게 따라오는 lag 효과

## 모바일 인터랙션

| 데스크탑 | 모바일 대응 |
|----------|------------|
| 마우스 흔들기 (캐러셀 이동) | 손가락 스와이프 |

## 데이터

### Spotify API
- 사용 데이터: 아티스트 이름, 이미지, 장르, 앨범 정보, 트랙 리스트, `external_urls.spotify`
- 아티스트 수: 30명 (팝 장르 기준 유명 아티스트)
- 포함 아티스트: 그린데이(Green Day), 레이니, 영블러드(Yungblud) 포함

### 수동 입력 데이터
- 대표곡 YouTube 링크
- 대표 앨범 YouTube 링크
- Instagram 링크

### Spotify API 법적 준수 사항
- 모든 페이지에 **"Data provided by Spotify"** 표기 필수
- Spotify 브랜딩 가이드라인 준수
- 오디오 스트리밍 불가 (Spotify Web Playback SDK 미사용)
- Spotify 데이터와 YouTube 링크는 완전히 분리 (수동 입력)
- 데이터 장기 캐싱 금지
